# 题库管理系统需求文档（重生成）

## 1. 文档说明
- 文档目的：基于当前项目代码重新整理系统需求。
- 代码基线：`D:\code\pratical_training\Question_Bank_Management_System`。
- 参考格式：对齐 `供gpt参考文件/diagram.md` 的模块化表达方式。

## 2. 项目目标
- 面向学生、教师、管理员提供统一的题库学习与教学管理平台。
- 支持从题目维护、组卷、发布作业，到学生作答、自动/人工评分、申诉复核、统计分析的完整闭环。
- 提供基于 LLM 的题目解析生成与主观题评分辅助能力。

## 3. 角色与权限
- 学生：作业/练习、作答、查看结果、查看统计、提交申诉、加入班级。
- 教师：标签管理、题目管理、试卷管理、作业管理、班级管理、复核评分、处理申诉、查看 LLM 调用记录。
- 管理员：具备教师能力，并可管理用户与系统日志（审计日志、登录日志）。

## 4. 功能范围

### 4.1 认证与账号
- 用户注册、登录、登出、获取当前登录用户信息。
- 基于 JWT 的无状态鉴权。
- RBAC 角色控制（`STUDENT`、`TEACHER`、`ADMIN`）。

### 4.2 标签管理
- 标签列表查询（支持关键词）。
- 标签树查询。
- 标签创建、修改、删除。

### 4.3 题目管理
- 题目增删改查、分页检索、多条件筛选（关键词/章节/难度/题型/状态/标签）。
- 题型覆盖：单选、多选、判断、填空、编程（代码题可配置样例）。
- 题目发布。
- LLM 生成题目解析。

### 4.4 试卷管理
- 试卷增删改查、分页。
- 向试卷增删题、设置题目顺序和分值。
- 试卷总分重算。

### 4.5 作业管理
- 作业增删改查、发布、关闭。
- 作业可配置开始/截止时间、时长、最大尝试次数、乱序策略。
- 作业目标可按学生 ID 或班级配置。

### 4.6 班级管理
- 教师创建班级、查看自己的班级、查看班级学生、移除学生。
- 学生通过班级码加入班级、查看我的班级。

### 4.7 作答与评分
- 学生开始作业作答或练习作答。
- 获取作答题目列表，支持草稿保存和提交单题答案。
- 提交整次作答并生成结果。
- 客观题自动评分；主观题进入 LLM/人工复核流程。

### 4.8 复核与申诉
- 学生提交申诉、查看我的申诉。
- 教师复核中心：待复核答案分页、证据查看、人工评分、LLM 重试。
- 教师处理申诉（同意/驳回及最终分数）。

### 4.9 统计分析
- 错题统计、错题消除。
- 标签掌握度统计。
- 用户能力分统计。
- 题目级作答统计。

### 4.10 管理后台
- 用户分页查询、创建用户、更新用户信息、更新用户角色。
- 审计日志分页查询。
- 登录日志分页查询。

## 5. 关键业务规则
- 注册/登录后，非公开接口需携带 JWT。
- 题目管理权限：教师仅可管理自己创建的题目，管理员可管理全部。
- 学生题目可见范围：已发布题目，且受教师可见范围和班级关系影响。
- 作业可见范围：未配置目标时全员可见；配置目标后仅目标学生/目标班级成员可见。
- 作业提交后进入评分阶段，客观题优先自动计分。
- 主观题评分支持 LLM 结果落库，低置信度或异常结果需人工复核。
- 申诉处理可更新最终分并形成评分记录。

## 6. 数据需求（逻辑层）
- 用户与角色：`sys_user`、`sys_role`、`sys_user_role`。
- 题库基础：`qb_tag`、`qb_question`、`qb_question_option`、`qb_question_tag`、`qb_question_case`。
- 组卷与作业：`qb_paper`、`qb_paper_question`、`qb_assignment`、`qb_assignment_target`、`qb_assignment_target_class`。
- 班级：`qb_class`、`qb_class_member`。
- 作答评分：`qb_attempt`、`qb_attempt_question`、`qb_answer`、`qb_grading_record`、`qb_appeal`、`qb_llm_call`。
- 学习统计：`qb_wrong_question`、`qb_tag_mastery`、`qb_user_ability`、`qb_question_user_stat`。
- 运维日志：`sys_audit_log`、`sys_login_log`。

## 7. 非功能需求
- 安全性：JWT + 方法级权限注解，密码 BCrypt 存储。
- 一致性：关键写操作使用事务，保证作业/作答/评分链路一致性。
- 可观测性：保留登录日志、审计日志与 LLM 调用记录。
- 可维护性：分层结构（Controller/Service/Mapper/Entity/DTO/VO）。
- 兼容性：前后端分离，REST API + JSON。

## 8. 本次明确不在范围
- 论坛模块（`qb_post`、`qb_reply`、`qb_post_tag`）已移除。
- OAuth/第三方登录未启用。
- 自动化部署与多租户能力不在当前实现范围。

## 9. 验收要点
- 学生侧：可完成“开始作答 -> 提交 -> 查看结果 -> 提交申诉 -> 查看统计”闭环。
- 教师侧：可完成“标签/题目/试卷/作业/班级管理 + 复核处理”闭环。
- 管理侧：可完成“用户管理 + 日志查询”闭环。
- 关键数据对象可在数据库中形成完整关联并可追溯。
