## 1. 用户角色与权限 (User & Roles)

### 1.1 角色定义

- **Student（学生）**：浏览题库、开始练习/作业、提交答案、查看解析、错题集/练习记录、发帖回帖、对评分申诉。
    
- **Teacher（教师）**：管理标签、管理题目、组卷、发布作业/测试、查看成绩/批改记录、处理复核与申诉、参与答疑区。
    
- **Admin（管理员）**：系统级管理（至少包含登录/退出；可扩展用户/角色管理、配置管理）。

## 2. 功能需求 (Functional Requirements)

_注：接口/实现时建议按 FR 编号落地，便于论文撰写与最终验收。_

### 2.1 基础与系统模块

- **FR-01 登录/退出**
    
    - **输入**：`username` + `password`
        
    - **处理**：校验账号状态；签发 JWT；返回用户基本信息+角色。
        
    - **输出**：Token、Role列表、过期时间。
        
    - **异常**：密码错误/账号禁用/频繁失败触发限流（可选）。
        
- **FR-02 鉴权与 RBAC**
    
    - 所有受保护 API 必须校验 JWT 有效性。
        
    - 关键资源（题目 CRUD、组卷发布、复核处理等）按角色进行访问控制。
        
    - 建议支持细粒度权限（Permission，可选）以支持未来扩展。
        
- **FR-17 日志与审计 (建议)**
    
    - **变更审计**：题目/标签/试卷的变更应记录审计日志（操作人、时间、变更内容）。
        
    - **登录日志**：记录成功/失败、IP、UA（可选）。
        

### 2.2 教师管理模块

- **FR-03 标签管理 (Teacher)**
    
    - 创建/编辑/删除（软删除）标签；支持层级（`parent_id`）与排序。
        
    - 用途：题库检索、组卷规则、统计诊断、答疑区归类。
        
- **FR-04 题目管理 (Teacher)**
    
    - 支持题目新增/编辑/删除（软删）、发布/下架。
        
    - **字段要求**：类型、难度、章节、题干、标准答案、解析、选项（客观题）、标签绑定。
        
    - **状态管理**：`DRAFT` (草稿) / `PUBLISHED` (发布) / `ARCHIVED` (归档)。
        
    - **智能辅助**：支持在保存题目时触发“LLM生成初稿解析”（可选开关）。
        
- **FR-06 组卷 (Paper, Teacher)**
    
    - 教师创建试卷模板，选择题目并设置题序与分值。
        
    - 支持两种组卷方式：
        
        1. **手动选题**
            
        2. **规则组卷**（可选扩展：按标签覆盖/题型比例/难度分布）
            
    - 试卷需计算 `total_score`，并可校验是否为100或符合任意配置。
        
- **FR-07 发布作业/测试 (Assignment, Teacher)**
    
    - 将 Paper 发布为 Assignment，需配置：
        
        - `start_time` / `end_time`（截止时间）
            
        - `time_limit_min`（限时，0表示不限时）
            
        - `max_attempts`（次数限制）
            
        - `shuffle_questions` / `shuffle_options`（随机化）
            
    - 支持发布状态：`DRAFT` / `PUBLISHED` / `CLOSED`。
        
- **FR-12 人工复核 (Teacher)**
    
    - 教师在“待复核列表”查看：学生答案、LLM输出JSON、原始文本、系统判定原因（低置信度/分歧过大/解析失败）。
        
    - 教师可修改最终分数并填写复核说明，系统记录 `MANUAL GradingRecord`，并更新 `Answer.final_score`。
        

### 2.3 学生学习模块

- **FR-05 题库浏览与检索 (Student/Teacher)**
    
    - 按标签/章节/难度/题型/关键词组合检索。
        
    - 支持分页、排序（按更新时间/难度/热门等可扩展）。
        
    - **权限**：学生端只读；教师端可进入编辑。
        
- **FR-08 开始作答 (Attempt, Student)**
    
    - 入口类型：
        
        - **作业/测试**：`Attempt.type = ASSIGNMENT`
            
        - **练习入口**：`Attempt.type = PRACTICE`（随机/自适应）
            
    - 系统创建 Attempt 并生成 `AttemptQuestion` 列表（保证本次题序与分值可追溯）。
        
    - **限制校验**：若超过截止时间或次数用尽，则拒绝开始。
        
- **FR-09 作答保存与提交 (Answer)**
    
    - 支持作答过程保存草稿（`Answer.status = DRAFT`）。
        
    - 提交后锁定 Attempt（`Attempt.status = SUBMITTED`），禁止再次修改（除非教师开放重做）。
        
    - **记录**：提交时间、用时、每题答案内容、客观题自动得分。
        
- **FR-14 错题集与练习记录 (Student)**
    
    - **错题集**：按题目维度聚合（错题次数、最近错题时间、是否已掌握）。
        
    - **练习记录**：按 Attempt 维度查看（得分、用时、题目列表、解析）。
        
    - 支持按标签筛选错题，定位薄弱知识点。
        
- **FR-15 自适应练习 (扩展)**
    
    - 系统维护数据：
        
        - `TagMastery` (0~1)：按标签聚合得分动态更新。
            
        - `UserAbility` (0~100)：近期成绩加权统计。
            
    - 学生选择范围（标签/章节）后，系统生成固定总分（如100分）的练习卷：
        
        - 题型配比可配置。
            
        - 难度区间按 ability 映射（允许±1浮动）。
            
        - 权重抽题策略：薄弱优先 + 难度匹配 + 新颖性去重。
            

### 2.4 评分与申诉模块

- **FR-10 自动判分 (客观题)**
    
    - **选择/判断**：与标准答案比对得分。
        
    - **填空**（若按精确匹配）：可按“忽略空格/大小写”规则配置。
        
    - 形成 `GradingRecord` (`mode=AUTO`)，写入得分细节。
        
- **FR-11 LLM辅助评分 (主观题, 扩展)**
    
    - 针对简答/代码解释等主观题，系统构建 rubric（得分点/扣分规则）并调用 LLM。
        
    - 要求 LLM 以 JSON 返回：`score`, `deductions`, `reason`, `confidence`。
        
    - **一致性校验**：支持多次评分/多 Prompt，若分歧超过阈值 → Answer/Attempt 标记 `NEED_REVIEW`。
        
    - **证据保留**：全量保留 Prompt、模型版本、响应内容、解析结果。
        
- **FR-13 申诉复核 (Student → Teacher)**
    
    - 学生对已出分答案发起申诉，填写理由与补充说明（可附截图/附件）。
        
    - 教师处理：同意/驳回，并给出最终裁决与说明；必要时调整最终分。
        
    - 申诉全过程可追溯（Appeal 表 + 关联 GradingRecord）。
        

### 2.5 社区模块

- **FR-16 答疑区 (扩展)**
    
    - 发帖、回帖、按标签归类。
        
    - 帖子可关联题目 (`related_question_id`)，形成“题目→讨论”入口。
        
    - 支持基本的内容状态管理（正常/删除）。

## 3. 非功能需求 (NFR)

- **NFR-01 安全性**
    
    - JWT 鉴权；密码使用 BCrypt 哈希存储，不可明文。
        
    - RBAC 权限控制（至少角色级）。
        
    - **越权防护**：严格防止越权访问（如：学生不可调用教师 CRUD 接口）。
        
    - **关键审计**：发布、复核、改分操作需记录审计日志。
        
- **NFR-02 性能**
    
    - 题库检索必须分页；组合检索字段（tag, difficulty, type, chapter）建立索引。
        
    - 统计查询应避免全表扫描（必要时引入汇总表/缓存）。
        
- **NFR-03 可用性与容错**
    
    - **LLM 降级策略**：调用失败或返回非 JSON 时，系统应降级为“待复核/人工评分”，不阻塞整体出分。
        
    - **提交幂等性**：重复提交同一 Attempt 应安全处理。
        
- **NFR-04 可维护性**
    
    - 后端分层架构：Controller / Service / Repository。
        
    - 规范：统一返回体结构、统一异常处理、统一日志规范。
        
- **NFR-05 可扩展性**
    
    - 题型可扩展（如增加编程题评测、更多自定义题型）。
        
    - 标签体系可扩展（支持多层级、多类型、多课程扩展）。

## 4. 数据需求 (Data Requirements)

以下为核心实体与关系（高频、必须落库）：

- **用户与权限**：`User`, `Role`, `Permission` (可选)
    
- **题库核心**：`Tag`, `Question`, `Option`, `QuestionTag` (多对多)
    
- **试卷与作业**：
    
    - `Paper`, `PaperQuestion` (记录题序/分值/快照)
        
    - `Assignment` (发布/截止/限时/次数)
        
- **作答记录**：
    
    - `Attempt`, `AttemptQuestion` (一次作答的题单快照)
        
    - `Answer` (学生答案)
        
- **评分与证据**：`GradingRecord`, `LlmCall` (批改证据链)
    
- **业务流程**：`Appeal` (申诉)
    
- **统计与自适应**：`TagMastery`, `UserAbility`, `QuestionUserStat`, `WrongQuestion`
    
- **社区**：`Post`, `Reply`, `PostTag`
