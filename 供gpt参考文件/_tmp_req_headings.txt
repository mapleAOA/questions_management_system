⽤⼾⻆⾊与权限 (User & Roles)
⻆⾊定义
/ Student（学⽣）：浏览题库、开始练习/作业、提交答案、查看解析、错题集/练习记录、发帖回帖、对评分申诉。
/ Teacher（教师）：管理标签、管理题⽬、组卷、发布作业/测试、查看成绩/批改记录、处理复核与申诉、参与答疑区。
/ Admin（管理员）：系统级管理（⾄少包含登录/退出；可扩展⽤⼾/⻆⾊管理、配置管理）。

功能需求 (Functional Requirements)
注：接⼝/实现时建议按 FR 编号落地，便于论⽂撰写与最终验收。
基础与系统模块
/ FR-01 登录/退出
输⼊：username + password
处理：校验账号状态；签发 JWT；返回⽤⼾基本信息+⻆⾊。
输出：Token、Role列表、过期时间。
异常：密码错误/账号禁⽤/频繁失败触发限流（可选）。
/ FR-02 鉴权与 RBAC 所有受保护 API 必须校验 JWT 有效性。
关键资源（题⽬ CRUD、组卷发布、复核处理等）按⻆⾊进⾏访问控制。
建议⽀持细粒度权限（Permission，可选）以⽀持未来扩展。
/ FR-17 ⽇志与审计 (建议)
变更审计：题⽬/标签/试卷的变更应记录审计⽇志（操作⼈、时间、变更内容）。
登录⽇志：记录成功/失败、IP、UA（可选）。
教师管理模块
/ FR-03 标签管理 (Teacher)
创建/编辑/删除（软删除）标签；⽀持层级（parent_id）与排序。
⽤途：题库检索、组卷规则、统计诊断、答疑区归类。
FR-04 题⽬管理 (Teacher)
⽀持题⽬新增/编辑/删除（软删）、发布/下架。
字段要求：类型、难度、章节、题⼲、标准答案、解析、选项（客观题）、标签绑定。
状态管理：DRAFT (草稿) / PUBLISHED (发布) / ARCHIVED (归档)。智能辅助：⽀持在保存题⽬时触发“LLM⽣成初稿解析（” 可选开关）。
/ FR-06 组卷 (Paper, Teacher)
/教师创建试卷模板，选择题⽬并设置题序与分值。
/⽀持两种组卷⽅式：
⼿动选题
规则组卷（可选扩展：按标签覆盖/题型⽐例/难度分布）/试卷需计算 total_score，并可校验是否为100或符合任意配置。
FR-07 发布作业/测试 (Assignment, Teacher) 将 Paper 发布为 Assignment，需配置： start_time / end_time（截⽌时间） time_limit_min（限时，0表⽰不限时） max_attempts（次数限制） shuffle_questions / shuffle_options（随机化）/⽀持发布状态：DRAFT / PUBLISHED / CLOSED。
/ FR-12 ⼈⼯复核 (Teacher)
/教师在“待复核列表”查看：学⽣答案、LLM输出JSON、原始⽂本、系统判定原因（低置信度/分歧过⼤/解析失败）。
教师可修改最终分数并填写复核说明，系统记录 MANUAL GradingRecord，并更
新 Answer.final_score。
学⽣学习模块
/ FR-05 题库浏览与检索 (Student/Teacher)
按标签/章节/难度/题型/关键词组合检索。⽀持分⻚、排序（按更新时间/难度/热⻔等可扩展）。
权限：学⽣端只读；教师端可进⼊编辑。
/ FR-08 开始作答 (Attempt, Student)
⼊⼝类型：
作业/测试：Attempt.type = ASSIGNMENT 练习⼊⼝：Attempt.type = PRACTICE（随机/⾃适应）
/系统创建 Attempt 并⽣成 AttemptQuestion 列表（保证本次题序与分值可追溯）。
/限制校验：若超过截⽌时间或次数⽤尽，则拒绝开始。
/ FR-09 作答保存与提交 (Answer)
⽀持作答过程保存草稿（Answer.status = DRAFT）。提交后锁定 Attempt（Attempt.status = SUBMITTED），禁⽌再次修改（除⾮教师开放重做）。
/记录：提交时间、⽤时、每题答案内容、客观题⾃动得分。
/ FR-14 错题集与练习记录 (Student) 错题集：按题⽬维度聚合（错题次数、最近错题时间、是否已掌握）。
练习记录：按 Attempt 维度查看（得分、⽤时、题⽬列表、解析）。
⽀持按标签筛选错题，定位薄弱知识点。
FR-15 ⾃适应练习 (扩展) 系统维护数据：
TagMastery (0~1)：按标签聚合得分动态更新。
UserAbility (0~100)：近期成绩加权统计。
学⽣选择范围（标签/章节）后，系统⽣成固定总分（如100分）的练习卷：题型配⽐可配置。
难度区间按 ability 映射（允许±1浮动）。
权重抽题策略：薄弱优先 + 难度匹配 + 新颖性去重。
评分与申诉模块
/ FR-10 ⾃动判分 (客观题)
选择/判断：与标准答案⽐对得分。
填空（若按精确匹配）：可按“忽略空格/⼤⼩写”规则配置。
形成 GradingRecord ( mode=AUTO )，写⼊得分细节。
/ FR-11 LLM辅助评分 (主观题, 扩展)
针对简答/代码解释等主观题，系统构建 rubric（得分点/扣分规则）并调⽤ LLM。
要求 LLM 以 JSON 返回：score , deductions , reason , confidence。
⼀致性校验：⽀持多次评分/多 Prompt，若分歧超过阈值 → Answer/Attempt 标记
NEED_REVIEW。
证据保留：全量保留 Prompt、模型版本、响应内容、解析结果。
/ FR-13 申诉复核 (Student → Teacher)
学⽣对已出分答案发起申诉，填写理由与补充说明（可附截图/附件）。教师处理：同意/驳回，并给出最终裁决与说明；必要时调整最终分。
申诉全过程可追溯（Appeal 表 + 关联 GradingRecord）。
社区模块
/ FR-16 答疑区 (扩展)
发帖、回帖、按标签归类。
帖⼦可关联题⽬ ( related_question_id )，形成“题⽬→讨论”⼊⼝。
⽀持基本的内容状态管理（正常/删除）。

⾮功能需求 (NFR)
/ NFR-01 安全性
JWT 鉴权；密码使⽤ BCrypt 哈希存储，不可明⽂。
RBAC 权限控制（⾄少⻆⾊级）。
越权防护：严格防⽌越权访问（如：学⽣不可调⽤教师 CRUD 接⼝）。
关键审计：发布、复核、改分操作需记录审计⽇志。
/ NFR-02 性能题库检索必须分⻚；组合检索字段（tag, difficulty, type, chapter）建⽴索引。
统计查询应避免全表扫描（必要时引⼊汇总表/缓存）。
/ NFR-03 可⽤性与容错
/ LLM 降级策略：调⽤失败或返回⾮ JSON 时，系统应降级为“待复核/⼈⼯评分”，不阻塞整体出分。
/提交幂等性：重复提交同⼀ Attempt 应安全处理。
/ NFR-04 可维护性
后端分层架构：Controller / Service / Repository。
规范：统⼀返回体结构、统⼀异常处理、统⼀⽇志规范。
/ NFR-05 可扩展性
题型可扩展（如增加编程题评测、更多⾃定义题型）。
标签体系可扩展（⽀持多层级、多类型、多课程扩展）。
