# 题库管理系统接口文档（重生成）

## 1. 基础说明
- 基础路径：`/api`
- 鉴权方式：`Authorization: Bearer <JWT>`
- 开放接口：`POST /api/register`、`POST /api/login`
- 其他接口：默认需登录，并结合 `@PreAuthorize` 做角色控制。

## 2. 通用响应

### 2.1 标准响应（除登录/注册）
```json
{
  "success": true,
  "msg": "ok",
  "code": null,
  "data": {}
}
```

### 2.2 分页对象
```json
{
  "page": 1,
  "size": 20,
  "total": 100,
  "list": []
}
```

### 2.3 登录/注册响应
`/api/register`、`/api/login` 返回 `LoginResponse`（不包裹 `ApiResponse`）：
```json
{
  "success": true,
  "token": "jwt-token",
  "user": {
    "id": 1,
    "username": "admin",
    "displayName": "系统管理员",
    "email": "admin@example.com",
    "roles": ["ADMIN"]
  },
  "msg": "ok"
}
```

## 3. 状态值字典
- 角色：`STUDENT`、`TEACHER`、`ADMIN`
- 题目状态：`1=draft, 2=published, 3=archived`
- 题型：`1=single, 2=multiple, 3=true_false, 4=blank, 5=short(停用), 6=code, 7=code_reading(停用)`
- 作业发布状态：`1=draft, 2=published, 3=closed`
- 作答类型：`1=assignment, 2=practice`
- 作答状态：`1=in_progress, 2=submitted, 3=grading, 4=graded`
- 答案状态：`1=draft, 2=submitted`
- 评分模式：`1=auto, 2=llm, 3=manual`
- 申诉状态：`1=pending, 2=approved, 3=rejected, 4=resolved`

## 5. 请求体结构

## 5.1 Auth
### RegisterRequest
- `username: String` 必填
- `password: String` 必填
- `role: String` 必填
- `displayName: String` 可选
- `email: String` 可选

### LoginRequest
- `username: String` 必填
- `password: String` 必填

## 5.2 标签与题目
### TagCreateRequest / TagUpdateRequest
- `tagName: String` 必填
- `tagCode: String` 可选
- `parentId: Long` 可选
- `tagLevel: Integer` 可选
- `tagType: Integer` 必填（1知识点/2章节/3自定义）
- `sortOrder: Integer` 可选

### QuestionOptionDTO
- `id: Long` 可选
- `optionLabel: String` 必填
- `optionContent: String` 必填
- `isCorrect: Integer` 可选（0/1）
- `sortOrder: Integer` 可选

### QuestionCaseDTO
- `id: Long` 可选
- `caseNo: Integer` 可选
- `inputData: String` 可选
- `expectedOutput: String` 可选
- `caseScore: Integer` 可选
- `isSample: Integer` 可选（0/1）

### QuestionUpsertRequest
- `title: String` 必填
- `questionType: Integer` 必填
- `difficulty: Integer` 必填（1-5）
- `chapter: String` 可选
- `stem: String` 必填
- `standardAnswer: String` 可选
- `answerFormat: Integer` 可选（默认1）
- `analysisText: String` 可选
- `analysisSource: Integer` 可选（默认1）
- `status: Integer` 可选（默认1）
- `options: List<QuestionOptionDTO>` 可选
- `tagIds: List<Long>` 可选
- `cases: List<QuestionCaseDTO>` 可选

## 5.3 试卷与作业
### PaperUpsertRequest
- `paperTitle: String` 必填
- `paperDesc: String` 可选
- `paperType: Integer` 必填（1手工/2规则）
- `status: Integer` 可选（默认1）
- `ruleJson: String` 可选

### PaperAddQuestionRequest
- `questionId: Long` 必填
- `orderNo: Integer` 必填
- `score: Integer` 必填

### PaperQuestionUpdateRequest
- `orderNo: Integer` 必填
- `score: Integer` 必填

### AssignmentUpsertRequest
- `paperId: Long` 必填
- `assignmentTitle: String` 必填
- `assignmentDesc: String` 可选
- `startTime: LocalDateTime` 可选
- `endTime: LocalDateTime` 必填
- `timeLimitMin: Integer` 可选（默认0）
- `maxAttempts: Integer` 可选（默认1）
- `shuffleQuestions: Integer` 可选（默认0）
- `shuffleOptions: Integer` 可选（默认0）
- `publishStatus: Integer` 可选（默认1）

### AssignmentTargetsRequest
- `userIds: List<Long>` 可选
- `classIds: List<Long>` 可选

## 5.4 班级与作答
### ClassCreateRequest
- `className: String` 必填
- `classDesc: String` 可选

### JoinClassRequest
- `classCode: String` 必填

### PracticeStartRequest
- `mode: String` 必填
- `scope.tagIds: List<Long>` 可选
- `scope.chapters: List<String>` 可选
- `scope.questionTypes: List<Integer>` 可选
- `scope.questionIds: List<Long>` 可选
- `totalScore: Integer` 可选（默认100）
- `ruleId: Long` 可选

### SaveAnswerDraftRequest
- `answerContent: String` 可选

## 5.5 申诉与复核
### AppealCreateRequest
- `answerId: Long` 必填
- `reasonText: String` 必填
- `attachments: List<String>` 可选

### AppealHandleRequest
- `action: String` 必填
- `finalScore: Integer` 可选（>=0）
- `decisionComment: String` 可选

### TeacherGradeRequest
- `score: Integer` 必填（>=0）
- `comment: String` 可选

### TeacherLlmRetryRequest
- `modelName: String` 可选
- `temperature: Double` 可选
- `times: Integer` 可选（1-5，默认1）

## 5.6 管理员
### AdminCreateUserRequest
- `username: String` 必填
- `password: String` 必填
- `displayName: String` 可选
- `email: String` 可选
- `status: Integer` 可选（默认1）
- `roles: List<String>` 必填

### AdminUpdateUserRequest
- `displayName: String` 可选
- `email: String` 可选
- `status: Integer` 可选

### AdminUpdateUserRolesRequest
- `roles: List<String>` 必填
